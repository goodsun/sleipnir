## 主な活動

- マスターからの依頼で、X（Twitter）への自動投稿システムの再構築を支援。
- Teddyの自動投稿システムがMac mini移行後に動作しなくなった問題の解決を試みる。
- 自動投稿方式として、以下の方法を検討・検証
    - Playwright
    - AppleScript + ネイティブ Chrome/Safari
    - OpenClaw browserツール
    - X API (v2)
- macOSのセキュリティポリシーにより、リモート接続からのAppleScriptのダイアログ表示が難しいことを確認。
- 最終的に、Safari + AppleScriptによる自動投稿システムを構築。
    - Safariのプロファイル機能を利用して、複数アカウント（Teddy, Akiko, goodsun, FLOW）への投稿を可能に。
    - 投稿には`⌘+Return`キーボードショートカットを使用。
    - 投稿スクリプト (`post_x.sh`) と note監視スクリプト (`note_watcher.py`) を作成し、Jasmine環境に保存。
    - 自動投稿のスケジュール管理にlaunchdを使用。
- goodsunアカウントでの長文投稿戦略を検討し、以下のアイデアを検証
    - Hacker News のトレンド記事を元にした長文投稿
    - 古典の笑い話や百人一首を現代風にアレンジした投稿
- FLOWアカウントでのダンマパダ現代語訳の投稿システムを構築。
- 投稿失敗時のリカバリ策として、Telegramへのintent URL通知機能を実装。
- Gemini APIの利用料金が無視できるほど低いことを確認。
- 各アカウントの投稿スケジュールを設定

## 決定事項・学び

- Xへの自動投稿には、bot検知のリスクを避けるため、ネイティブブラウザ (Safari) + AppleScript の組み合わせが最も安全。
- macOSのセキュリティ制限により、リモート環境からGUI操作を伴う自動化は困難。
- Safariのプロファイル機能を活用することで、複数アカウントへの自動投稿が可能。
- URLエンコードされたテキストの文字数制限により、長文投稿には工夫が必要。
- 投稿失敗時のリカバリ策として、Telegramへの通知が有効。
- OpenClawのエージェント呼び出し (`openclaw agent`) は対話形式であり、スクリプトからの利用には適さない。
- 今後のタスクは、必要に応じてプロンプトを調整し、投稿内容の質を向上させること。
